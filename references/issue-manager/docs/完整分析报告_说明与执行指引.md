## 报告输出位置

| 报告类型 | 路径 |
|----------|------|
| 日报 | `data/reports/daily_report_matrixorigin_matrixone_YYYYMMDD.json` |
| 进度报告 | `data/reports/progress_report_matrixorigin_matrixone_YYYYMMDD.json` |
| 多维度综合 | `data/reports/comprehensive_report_YYYYMMDD.json` |
| 客户报告 | `data/reports/customer_reports/` |
| 共用 Feature | `data/reports/shared_features_YYYYMMDD.json` |
| 风险分析 | `data/reports/risk_analysis_YYYYMMDD.json` |
| 可扩展分析（JSON/MD） | `data/reports/extensible_analysis_YYYYMMDD.json` / `.md` |
| AI 驱动分析 | `data/reports/ai_analysis/project_progress_YYYYMMDD.json`、`cross_patterns_YYYYMMDD.json` |

可扩展分析由 `config/analysis_config.yaml` 配置，支持基础统计、标签分析、模块分析、层级分析、客户分析、关联分析、趋势分析等维度。使用 `--extensible-report` 运行 `auto_run.py` 并发送邮件时，邮件正文会展示这些分析结果（标签、模块、客户、基础统计等）。

使用 `--ai-report` 时，会**同时分析** matrixone 和 matrixflow，将项目推进（客户健康度、关键发现、建议行动）和横向关联分析写入邮件正文，**matrixflow 排在 matrixone 前面**。
