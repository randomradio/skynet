## 📋 系统介绍

### 这个系统是做什么的？

这个系统可以自动从GitHub获取项目的Issue（问题/任务），使用AI智能分析每个Issue的类型、优先级、标签等信息，然后保存到数据库中，最后生成各种分析报告。

### 主要功能

1. **自动采集**：从GitHub自动获取所有Issue数据
2. **智能分析**：使用AI自动分类、提取优先级和标签
3. **数据存储**：保存到数据库（支持多种数据库）
4. **报告生成**：自动生成日报、进度报告、多维度分析报告、可扩展分析报告

## 🛠️ 准备工作

### 需要安装的软件

#### 1. Python（必须）

**什么是Python？**
Python是一种编程语言，我们的系统是用Python写的，所以需要安装它。

**如何安装？**

**Windows系统：**
1. 打开浏览器，访问：https://www.python.org/downloads/
2. 点击"Download Python 3.12"（或最新版本）
3. 下载完成后，双击安装文件
4. **重要**：安装时一定要勾选"Add Python to PATH"（添加到路径）
5. 点击"Install Now"（立即安装）
6. 等待安装完成

**Mac系统：**
1. 打开"终端"（Terminal）应用
2. 输入命令：`brew install python3`（如果没有brew，先安装Homebrew）
3. 或者访问 https://www.python.org/downloads/ 下载安装包

**如何检查是否安装成功？**
1. 打开"终端"（Windows是"命令提示符"或"PowerShell"）
2. 输入：`python3 --version` 或 `python --version`
3. 如果显示版本号（如：Python 3.12.0），说明安装成功

#### 2. 数据库（选择一个）

**选项A：SQLite（最简单，推荐新手）**
- 不需要单独安装，Python自带
- 数据保存在本地文件
- 适合测试和小型项目

**选项B：PostgreSQL（功能强大）**
- 需要单独安装
- Windows：https://www.postgresql.org/download/windows/
- Mac：`brew install postgresql`

**选项C：MySQL（常用）**
- 需要单独安装
- Windows：https://dev.mysql.com/downloads/installer/
- Mac：`brew install mysql`

**选项D：MatrixOne（如果使用）**
- 按照MatrixOne官方文档安装

### 需要准备的信息

1. **GitHub Token（必须）**
   - 登录GitHub
   - 访问：https://github.com/settings/tokens
   - 点击"Generate new token" → "Generate new token (classic)"
   - 填写名称（如：Issue Manager）
   - 勾选权限：`repo`（全部）
   - 点击"Generate token"
   - **重要**：复制生成的token，保存好（只显示一次）

2. **AI API密钥（必须选择一个）**
   
   **选项A：OpenAI（推荐）**
   - 访问：https://platform.openai.com/api-keys
   - 注册/登录账号
   - 创建API密钥
   - 复制密钥保存好
   
   **选项B：Claude**
   - 访问：https://console.anthropic.com/
   - 注册/登录账号
   - 创建API密钥
   
   **选项C：通义千问**
   - 访问：https://dashscope.aliyuncs.com/
   - 注册/登录账号
   - 创建API密钥

## 📦 安装步骤

### 第一步：下载代码

1. 确保代码文件夹在桌面：`GitHub_Issue_智能管理系统`
2. 如果不在，请将整个文件夹复制到桌面

### 第二步：打开终端

**Windows：**
1. 按 `Win + R` 键
2. 输入 `cmd` 或 `powershell`，按回车

**Mac：**
1. 按 `Command + 空格键`
2. 输入 `terminal`，按回车

### 第三步：进入项目文件夹

在终端中输入以下命令（根据你的实际路径调整）：

```bash
cd ~/Desktop/GitHub_Issue_智能管理系统
```

**Windows用户注意：**
如果路径有中文，可能需要使用：
```bash
cd C:\Users\你的用户名\Desktop\GitHub_Issue_智能管理系统
```

### 第四步：安装依赖包

在终端中输入：

```bash
pip3 install -r requirements.txt
```

或者：

```bash
python3 -m pip install -r requirements.txt
```

**如果提示权限错误：**
- Windows：在命令前加 `python -m`
- Mac/Linux：在命令前加 `sudo`

**等待安装完成**（可能需要几分钟）

## ⚙️ 配置说明

### 配置文件位置

所有配置都在：`config/config.py` 文件中

### 如何修改配置？

#### 方法一：直接编辑文件（推荐）

1. 用文本编辑器打开 `config/config.py` 文件
   - Windows：可以用"记事本"或"VS Code"
   - Mac：可以用"文本编辑"或"VS Code"

2. 找到需要修改的地方，直接修改：

```python
# GitHub Token（第15行左右）
GITHUB_TOKEN = "你的GitHub_Token_粘贴在这里"

# AI配置（第25行左右）
AI_PROVIDER = "openai"  # 改为你使用的AI服务：openai, claude, qwen
OPENAI_API_KEY = "你的OpenAI_API_Key_粘贴在这里"

# 数据库配置（第45行左右）
DATABASE_TYPE = "sqlite"  # 改为你使用的数据库：sqlite, postgresql, mysql, matrixone
```

#### 方法二：使用环境变量（高级）

如果你熟悉环境变量，可以设置：

```bash
export GITHUB_TOKEN="你的token"
export OPENAI_API_KEY="你的key"
```

### 配置项详细说明

#### 1. GitHub配置

```python
GITHUB_TOKEN = "ghp_xxxxxxxxxxxx"  # 你的GitHub Token
```

#### 2. AI配置

```python
# 选择AI服务提供商
AI_PROVIDER = "openai"  # 可选：openai, claude, qwen, local

# OpenAI配置
OPENAI_API_KEY = "sk-xxxxxxxxxxxx"
OPENAI_BASE_URL = "https://api.openai.com/v1"  # 一般不需要改
OPENAI_MODEL = "gpt-3.5-turbo"  # 可以改为 gpt-4

# Claude配置（如果使用Claude）
CLAUDE_API_KEY = "sk-ant-xxxxxxxxxxxx"
CLAUDE_MODEL = "claude-3-sonnet-20240229"

# 通义千问配置（如果使用通义千问）
QWEN_API_KEY = "sk-xxxxxxxxxxxx"
QWEN_MODEL = "qwen-turbo"
```

#### 3. 数据库配置

**SQLite（最简单）：**
```python
DATABASE_TYPE = "sqlite"
# SQLite不需要其他配置，数据会保存在 data/github_issues.db
```

**PostgreSQL：**
```python
DATABASE_TYPE = "postgresql"
POSTGRESQL_HOST = "localhost"  # 数据库地址
POSTGRESQL_PORT = "5432"  # 端口
POSTGRESQL_USER = "postgres"  # 用户名
POSTGRESQL_PASSWORD = "你的密码"  # 密码
POSTGRESQL_DATABASE = "github_issues"  # 数据库名
```

**MySQL/MatrixOne：**
```python
DATABASE_TYPE = "mysql"  # 或 "matrixone"
MYSQL_HOST = "localhost"
MYSQL_PORT = "3306"  # MatrixOne可能是6001
MYSQL_USER = "root"
MYSQL_PASSWORD = "你的密码"
MYSQL_DATABASE = "github_issues"
```

#### 4. 全量重新运行配置

如果你想清空旧数据，重新采集所有数据：

```python
ENABLE_FULL_RESYNC = True  # 改为True
```

**注意**：这会删除所有旧数据，请谨慎使用！

## 🚀 运行系统

### 方法一：使用运行脚本（最简单）

1. 双击 `run.py` 文件
2. 或者在终端输入：
```bash
python3 run.py
```

### 方法二：直接运行主程序

在终端输入：
```bash
python3 main.py
```

### 运行流程

1. **系统会检查配置**
   - 如果配置有问题，会提示你修改

2. **输入仓库信息**
   ```
   请输入仓库所有者（例如：octocat）: 你的用户名
   请输入仓库名称（例如：Hello-World）: 你的仓库名
   ```

3. **选择同步模式**
   ```
   是否执行全量同步（清空旧数据）？(y/N): 
   ```
   - 输入 `y`：清空旧数据，重新采集所有数据
   - 输入 `n` 或直接回车：增量同步，只采集新数据

4. **等待处理**
   - 系统会自动：
     - 从GitHub获取Issues
     - 使用AI分析每个Issue
     - 保存到数据库
     - 提取关联关系

5. **生成报告**
   ```
   是否生成报告？(Y/n): 
   ```
   - 输入 `y` 或直接回车：生成报告
   - 报告保存在 `data/reports/` 文件夹

## 📚 各模块说明

### 模块1：GitHub采集模块

**位置**：`modules/github_collector/github_api.py`

**功能**：
- 从GitHub API获取Issue数据
- 获取评论和时间线
- 提取Issue之间的关联关系

**输入**：
- 仓库所有者（owner）
- 仓库名称（repo）
- Issue状态（state）

**输出**：
- Issue数据列表
- 评论列表
- 关联关系列表

**使用方法**：
```python
from modules.github_collector.github_api import GitHubCollector

collector = GitHubCollector()
issues = collector.fetch_issues("owner", "repo", state="all")
```

---

### 模块2：LLM解析模块

**位置**：`modules/llm_parser/llm_parser.py`

**功能**：
- 使用AI分类Issue类型（bug/feature/task）
- 提取优先级（P0/P1/P2/P3）
- 提取标签
- 生成摘要
- 分析阻塞原因

**输入**：
- Issue标题
- Issue正文
- 评论列表（可选）

**输出**：
- 分类结果：`{"type": "bug", "confidence": 0.9}`
- 优先级：`"P1"`
- 标签列表：`["前端", "UI", "bug"]`
- 摘要文本
- 阻塞原因（如果有）

**使用方法**：
```python
from modules.llm_parser.llm_parser import LLMParser

parser = LLMParser()
result = parser.classify_issue("修复登录问题", "用户无法登录")
# 输出: {"type": "bug", "confidence": 0.85}
```

**支持的AI服务**：
- OpenAI（GPT-3.5, GPT-4）
- Claude（Claude 3）
- 通义千问
- 本地模型（OpenAI兼容接口）

**如何切换AI服务？**
在 `config/config.py` 中修改：
```python
AI_PROVIDER = "openai"  # 改为 claude, qwen, local
```

---

### 模块3：数据库存储模块

**位置**：`modules/database_storage/mo_client.py`

**功能**：
- 保存Issue快照到数据库
- 保存评论
- 保存关联关系（**已修复Issue Number到Issue ID转换问题**）
- 支持全量重新运行

**输入**：
- Issue数据字典
- 评论列表
- 关联关系列表

**输出**：
- 数据保存到数据库
- 返回保存结果

**使用方法**：
```python
from modules.database_storage.mo_client import MOStorage

storage = MOStorage()
storage.save_issue_snapshot(issue_data, datetime.now())
```

**支持的数据库**：
- SQLite
- PostgreSQL
- MySQL
- MatrixOne

**如何切换数据库？**
在 `config/config.py` 中修改：
```python
DATABASE_TYPE = "sqlite"  # 改为 postgresql, mysql, matrixone
```

**全量重新运行**：
```python
# 在config/config.py中设置
ENABLE_FULL_RESYNC = True

# 或者在代码中调用
storage.clear_all_data()
```

---

### 模块4：报告生成模块

**位置**：`modules/report_generator/report_gen.py`

**功能**：
- 生成日报
- 生成进度报告
- 统计各种指标

**输入**：
- 仓库所有者
- 仓库名称
- 目标日期（可选）

**输出**：
- JSON格式的报告文件
- 保存在 `data/reports/` 目录

**使用方法**：
```python
from modules.report_generator.report_gen import Analyzer
from modules.database_storage.mo_client import MOStorage

storage = MOStorage()
analyzer = Analyzer(storage)
report = analyzer.generate_daily_report("owner", "repo")
```

**报告内容**：
- 总Issue数
- 开放/关闭数量
- 新增/关闭数量（今日）
- 阻塞Issue数量
- 按状态/类型/优先级统计

### 模块5：可扩展分析引擎

**位置**：`modules/analysis_extensible/`

**功能**：配置驱动的多维度分析，支持基础统计、标签分析、模块分析、层级分析、客户分析、关联分析、趋势分析等，输出 JSON/Markdown/HTML。

**配置**：`config/analysis_config.yaml`

**使用方法**：
```bash
python3 run_extensible_analysis.py --repo-owner matrixorigin --repo-name matrixone
```
或在 `auto_run.py` 中加 `--extensible-report` 参数。

## ❓ 常见问题

### Q1: 提示"配置验证失败"

**原因**：GitHub Token或AI API Key未设置

**解决方法**：
1. 打开 `config/config.py`
2. 找到 `GITHUB_TOKEN` 和 `OPENAI_API_KEY`（或其他AI的Key）
3. 将你的Token/Key粘贴进去
4. 保存文件

---

### Q2: 提示"数据库连接失败"

**原因**：数据库配置错误或数据库未启动

**解决方法**：

**SQLite用户**：
- 检查 `data` 文件夹是否存在
- 如果不存在，系统会自动创建

**PostgreSQL/MySQL用户**：
1. 确认数据库服务已启动
2. 检查 `config/config.py` 中的数据库配置：
   - 主机地址（host）
   - 端口（port）
   - 用户名（user）
   - 密码（password）
   - 数据库名（database）
3. 确认数据库已创建（如果不存在，需要先创建）

**测试数据库连接**：
```python
python3 -c "from config.config import get_database_url; print(get_database_url())"
```

---

### Q3: 提示"AI调用错误"

**原因**：
1. AI API Key错误或过期
2. API余额不足
3. 网络问题

**解决方法**：
1. 检查API Key是否正确
2. 登录AI服务网站，检查余额
3. 检查网络连接
4. 如果使用本地模型，确认服务已启动

**回退机制**：
系统有基于规则的回退方法，即使AI调用失败，也能继续工作（但准确度会降低）

---

### Q4: 提示"无法找到 Issue #123 的 ID"

**原因**：关联关系中的目标Issue还未被采集

**解决方法**：
- 这是正常情况，系统会跳过这些关系
- 如果目标Issue后续被采集，关系会自动建立
- 或者执行全量同步，确保所有Issue都被采集

---

1. 打开 `config/config.py`

   ```python
   AI_PROVIDER = "claude"  # 改为你想要的AI服务
   ```

### Q7: 如何更换数据库？

1. 打开 `config/config.py`
2. 修改 `DATABASE_TYPE`：
   ```python
   DATABASE_TYPE = "postgresql"  # 改为你想要的数据库
   ```
3. 设置对应的数据库配置
4. 安装对应的驱动（如果需要）：

---

### Q8: 报告文件在哪里？

报告文件保存在：`data/reports/` 文件夹

文件名格式：
- 日报：`daily_report_所有者_仓库名_日期.json`
- 进度报告：`progress_report_所有者_仓库名_日期.json`
- 多维度报告：`comprehensive_report_日期.json`、`customer_reports/` 等
- 可扩展分析：`extensible_analysis_日期.json`、`extensible_analysis_日期.md`

可以用文本编辑器打开查看，或者用JSON查看器格式化显示。可扩展分析支持独立运行：`python3 run_extensible_analysis.py --repo-owner 所有者 --repo-name 仓库名`。使用 `--extensible-report` 运行 `auto_run.py` 并发送邮件时，邮件正文会包含综合分析。诊断脚本：`python3 test_analysis.py`。

---

### Q9: 系统运行很慢怎么办？

**可能原因**：
1. Issue数量太多
2. AI API调用慢
3. 网络问题

**优化建议**：
1. 使用增量同步，只采集新数据
2. 使用更快的AI模型（如GPT-3.5-turbo）
3. 检查网络连接
4. 分批处理（修改代码，每次处理一部分）

---

### Q10: 如何查看日志？

系统会在终端输出运行日志。

如果需要保存日志：
1. 运行程序时使用重定向：
   ```bash
   python3 main.py > logs/run.log 2>&1
   ```
2. 日志会保存在 `logs/run.log` 文件
