## 二、发现的问题

### 问题1：`issue_relations` 表无 `repo_owner`、`repo_name` 列 ⚠️ 严重

**现状**：现有项目的 `issue_relations` 表结构为：
```sql
issue_relations (from_issue_id, to_issue_id, relation_type, ...)
-- 没有 repo_owner, repo_name
```

**优化方案中的 SQL**（会报错）：
```sql
SELECT from_issue_id, to_issue_id, relation_type
FROM issue_relations
WHERE repo_owner = :owner AND repo_name = :repo  -- ❌ 列不存在
```

**影响**：`find_shared_features()` 和 `analyze_blocking_chains()` 会执行失败。

**修复**：改为通过 `issues_snapshot` 关联过滤，只保留该仓库内的 relation。

---

### 问题2：`improved_report_generator` 导入路径

**现状**：`from modules.analysis_engine.multi_dimensional_analyzer import MultiDimensionalAnalyzer`

**要求**：`multi_dimensional_analyzer.py` 必须位于 `modules/analysis_engine/` 目录。

**结论**：按文档放置即可，需创建 `modules/analysis_engine/` 目录。

---

### 问题3：报告输出目录不统一

**现状**：`improved_report_generator` 默认 `output_dir="reports"`，项目使用 `data/reports`。

**修复**：使用 `config.REPORT_OUTPUT_DIR` 或 `data/reports`，与现有报告保持一致。

---

### 问题4：`multi_dimensional_analyzer` 的 sys.path

**现状**：`sys.path.insert(0, os.path.join(os.path.dirname(__file__), '../../'))`

**结论**：文件在 `modules/analysis_engine/` 时，`../../` 指向项目根目录，导入正确。
