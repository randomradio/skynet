# 快速开始指南

5分钟快速上手GitHub Issue智能管理系统！

## ⚡ 三步快速开始

### 第一步：安装依赖（1分钟）

```bash
cd "/Users/wupeng/Desktop/GitHub_Issue_智能管理系统"
pip3 install -r requirements.txt
```

### 第二步：配置系统（2分钟）

打开 `config/config.py`，修改以下配置：

```python
# 1. GitHub Token（必须）
GITHUB_TOKEN = "ghp_你的GitHub_Token_粘贴这里"

# 2. AI配置（必须，选择一个）
AI_PROVIDER = "openai"  # 或 "claude", "qwen"
OPENAI_API_KEY = "sk-你的OpenAI_Key_粘贴这里"

# 3. 数据库配置（已配置MatrixOne，可直接使用）
DATABASE_TYPE = "matrixone"  # 已配置，可直接使用
# 如需使用SQLite（本地测试），可改为 "sqlite"
```

**如何获取Token/Key？**

- **GitHub Token**：https://github.com/settings/tokens → Generate new token → 勾选 `repo` 权限
- **OpenAI Key**：https://platform.openai.com/api-keys → Create new secret key
- **Claude Key**：https://console.anthropic.com/ → API Keys → Create Key
- **通义千问Key**：https://dashscope.aliyuncs.com/ → API密钥管理

### 第三步：运行系统（2分钟）

```bash
python3 run.py
```

然后按提示输入：
1. 仓库所有者（如：`microsoft`）
2. 仓库名称（如：`vscode`）
3. 是否全量同步（输入 `n` 或直接回车）
4. 是否生成报告（输入 `y` 或直接回车）

**完成！** 🎉

**注意**：系统已配置 MatrixOne 数据库，首次运行时会自动创建数据库表。如果遇到连接问题，请参考 [MatrixOne配置指南](../MatrixOne配置指南.md)。

## 📋 最小配置示例

如果你只想快速测试，这是最小配置：

```python
# config/config.py 最小配置

# GitHub（必须）
GITHUB_TOKEN = "ghp_xxxxxxxxxxxx"

# AI（必须，选择一个）
AI_PROVIDER = "openai"
OPENAI_API_KEY = "sk-xxxxxxxxxxxx"

# 数据库（默认SQLite，无需配置）
DATABASE_TYPE = "sqlite"
```

其他配置项都有默认值，可以暂时不设置。

## 🎯 第一次运行

第一次运行时，系统会：

1. ✅ 检查配置是否正确
2. 📥 从GitHub获取所有Issues
3. 🤖 使用AI分析每个Issue
4. 💾 保存到数据库（`data/github_issues.db`）
5. 📊 生成报告（`data/reports/`）

**预计时间**：
- 小型仓库（<100 Issues）：5-10分钟
- 中型仓库（100-1000 Issues）：10-30分钟
- 大型仓库（>1000 Issues）：30分钟以上

## 🔍 验证是否成功

运行完成后，检查：

1. **数据库文件**：`data/github_issues.db` 应该存在
2. **报告文件**：`data/reports/` 文件夹中应该有JSON报告
3. **终端输出**：应该显示"✅ 所有任务完成！"

## ❓ 遇到问题？

### 问题1：`pip3: command not found`

**解决**：
- Mac：`python3 -m pip install -r requirements.txt`
- Windows：`python -m pip install -r requirements.txt`

### 问题2：配置验证失败

**解决**：检查Token和Key是否正确粘贴，注意不要有多余的空格

### 问题3：AI调用失败

**解决**：
- 检查API Key是否正确
- 检查账户余额
- 检查网络连接

### 问题4：数据库连接失败

**解决**：
- SQLite：确保 `data` 文件夹存在（系统会自动创建）
- PostgreSQL/MySQL：确认数据库服务已启动
