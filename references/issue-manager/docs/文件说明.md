# 项目文件说明

## 📁 根目录（主程序与入口）

仅保留核心入口和必要配置，便于快速定位：

- `main.py` - 主程序入口
- `run.py` - 快速运行（调用 main）
- `auto_run.py` - 自动运行（定时任务，推荐）
- `config.example.json` - 配置示例
- `create_tables.sql` - 建表 SQL
- `requirements.txt` - 依赖列表
- `README.md` - 项目总览

## 📂 scripts/ 辅助脚本

所有辅助脚本归档在 `scripts/`，详见 [scripts/README.md](../scripts/README.md)：

- `supplement_relations.py` - 关联关系补全
- `repair_ai_parse.py` - AI 解析补全
- `run_full_repair_and_analysis.py` - 补全 + 完整分析报告
- `run_analysis.py` - 分析运行（日报/进度报告/邮件）
- `run_extensible_analysis.py` - 可扩展分析（标签/模块/趋势等）
- `run_ai_analysis.py` - AI 驱动分析（项目推进 + 横向关联，使用配置的 AI 如千问）
- `check_config.py` - 配置检查
- `check_and_send.py` - 配置检查与邮件发送
- `test_db_connection.py` - 数据库连接测试
- `query_labels_sample.py` - Labels 查询样例
- `migrate_issue_id_to_bigint.py` - 数据库迁移
- `一键补全并发送报告.sh` - 一键补全并发送快捷脚本

## 📂 config/ 配置

- `config/config.py` - 主配置文件
- `config/analysis_config.yaml` - 可扩展分析配置

## 📂 modules/ 功能模块

- `modules/github_collector/` - GitHub 数据采集
- `modules/llm_parser/` - AI 智能分析
- `modules/database_storage/` - 数据库存储
- `modules/report_generator/` - 报告生成
- `modules/analysis_engine/` - 多维度分析
- `modules/analysis_extensible/` - 可扩展分析引擎
- `modules/email_sender/` - 邮件发送
