# 项目清理和自动运行脚本说明

## 完成的工作

### 1. 创建可配置的自动运行脚本

创建了 `auto_run.py` 脚本，具有以下特点：

- **多种配置方式**：
  - 支持 JSON/YAML 配置文件
  - 支持环境变量
  - 支持命令行参数
  - 配置优先级：命令行参数 > 配置文件 > 环境变量 > 默认值

- **灵活的运行选项**：
  - `--skip-sync`: 跳过数据同步
  - `--skip-report`: 跳过报告生成
  - `--skip-email`: 跳过邮件发送
  - `--full-sync`: 全量同步（清空旧数据）
  - `--multi-report`: 额外生成多维度分析报告
  - `--extensible-report`: 额外生成可扩展分析报告（标签/模块/趋势等）
  - `--check-only`: 仅检查配置

- **完善的错误处理**：
  - 配置检查
  - 详细的错误信息
  - 异常处理和日志输出

## 使用建议

### 日常使用

- **交互式运行**：使用 `python3 run.py` 或 `python3 main.py`
- **快速测试**：使用 `python3 scripts/run_analysis.py --repo-owner OWNER --repo-name REPO`

### 定时任务

- **推荐使用**：`python3 auto_run.py --config config.json`
- 支持 crontab、Windows 任务计划、systemd 等多种定时任务方式

### 配置检查

- 使用 `python3 scripts/check_config.py` 进行详细的配置检查
- 使用 `python3 auto_run.py --check-only` 进行快速配置检查

## 配置文件示例

创建 `config.json`：

```json
{
  "repo_owner": "matrixorigin",
  "repo_name": "matrixone",
  "email": "wupeng@matrixorigin.cn",
  "full_sync": false,
  "skip_email": false,
  "skip_sync": false,
  "skip_report": false,
  "multi_report": false,
  "extensible_report": false
}
```

然后运行：

```bash
python3 auto_run.py --config config.json
```

## 定时任务示例

### Linux/macOS (crontab)

每天上午 9 点运行：

```bash
0 9 * * * cd /path/to/GitHub_Issue_智能管理系统 && /usr/bin/python3 auto_run.py --config config.json >> logs/auto_run.log 2>&1
```
