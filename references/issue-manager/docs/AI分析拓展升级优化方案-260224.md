## 一、方案概述

### 两条分析线

| 分析线 | 频率 | 用途 |
|--------|------|------|
| **项目推进分析** | 每日/每周 | 按客户、层级（L1 项目→L2 Feature→L3 Task→L4 Bug）分析进度，输出健康度、堵塞点、AI 建议 |
| **横向关联分析** | 每周/每月 | 共性 Feature、高 Bug Feature、跨客户模式、资源分配建议 |

### 与现有模块的关系

| 模块 | 状态 | 说明 |
|------|------|------|
| **可扩展分析** (`analysis_extensible`) | ✅ 已集成 | 配置驱动：标签、模块、层级、客户、关联、趋势；`--extensible-report` 已接入 auto_run |
| **多维度分析** (`improved_report_gen`) | ✅ 已集成 | 客户项目、共用 Feature、阻塞链；`--multi-report` 已接入 |
| **AI 驱动分析** (`ai_analysis`) | 🆕 可选 | 项目推进 + 横向关联分析，使用 config 配置的 AI（默认千问） |

## 二、可扩展分析框架（已实现）

### 架构

```
analysis_config.yaml → ExtensibleAnalysisEngine → Analyzers → Formatters (JSON/MD/HTML)
```

### 已启用的分析器

- `basic_stats` - 基础统计
- `label_analysis` - 标签分析（kind/、area/、customer/）
- `module_analysis` - 功能模块分析
- `hierarchy_analysis` - 层级分析
- `customer_analysis` - 客户维度
- `relation_analysis` - 关联分析
- `trend_analysis` - 时间趋势

### 使用方式

```bash
python3 auto_run.py --repo-owner matrixorigin --repo-name matrixone --extensible-report --email user@example.com
python3 scripts/run_extensible_analysis.py --repo-owner matrixorigin --repo-name matrixone
```

## 三、AI 驱动分析（可选能力）

### 能力说明

- **项目推进分析**：按客户构建层级、识别堵塞、调用 AI 输出健康度、关键发现、建议行动
- **横向关联分析**：共性 Feature、高 Bug Feature、AI 洞察与策略建议

### 依赖

- 使用 config 中配置的 AI（`AI_PROVIDER`，默认千问），与主系统一致
- 适用于 matrixflow 等含 `customer/xxx` 标签的仓库

### 使用方式

```bash
# 方式一：auto_run 集成（推荐，分析 matrixone + matrixflow，结果写入邮件，matrixflow 优先展示）
python3 auto_run.py --repo-owner matrixorigin --repo-name matrixflow --skip-sync --ai-report --email 收件人@example.com

# 方式二：独立运行（仅生成 JSON 报告，不发邮件）
python3 scripts/run_ai_analysis.py --repo-owner matrixorigin --repo-name matrixflow
```

### 输出

- `data/reports/ai_analysis/project_progress_YYYYMMDD.json`
- `data/reports/ai_analysis/cross_patterns_YYYYMMDD.json`（每周一）

### 邮件集成

- 使用 `auto_run.py --ai-report` 时，会**同时分析** `matrixorigin/matrixflow` 和 `matrixorigin/matrixone`
- 分析结果（项目推进 + 横向关联）插入邮件正文，在页脚前展示
- **邮件中 matrixflow 排在 matrixone 前面**（用户更关注 matrixflow）
- `email_sender.format_ai_analysis_section()` 支持多仓库结构：`by_repo` + `order`

## 五、配置与扩展

### analysis_config.yaml

修改 `config/analysis_config.yaml` 可调整可扩展分析的内容与输出格式，无需改代码。

### 新增分析器

继承 `BaseAnalyzer`，实现 `analyze(issues) -> Dict`，在配置中启用即可。

### 新增 AI 分析

`modules/ai_analysis/ai_driven_analysis_engine.py` 使用项目统一的 LLM 调用（`LLMParser`），自动跟随 config 中的 `AI_PROVIDER`（千问/Claude/OpenAI 等）。
