# 自动运行脚本使用说明

## 概述

`auto_run.py` 是一个可配置的自动运行脚本，用于自动化执行 GitHub Issue 智能管理系统的完整流程，包括：
- 数据同步（从 GitHub 获取 Issue 数据）
- 关联关系补全（可选，补充因顺序跳过的关联）
- 报告生成（生成日报和进度报告）
- 邮件发送（将报告发送到指定邮箱）
- **数据库备份**（可选，建议作为**单独定时事项**执行，参见下方「定时事项说明」）

## 功能特点

1. **多种配置方式**：支持配置文件、环境变量、命令行参数
2. **灵活的运行选项**：可以跳过某些步骤，只执行需要的部分
3. **配置检查**：运行前自动检查系统配置
4. **错误处理**：完善的错误处理和日志输出

## 安装要求

确保已安装所有依赖：

```bash
pip3 install -r requirements.txt
```

## 配置方式

### 方式1：使用配置文件（推荐）

创建配置文件 `config.json`：

```json
{
  "repo_owner": "matrixorigin",
  "repo_name": "matrixone",
  "email": "wupeng@matrixorigin.cn",
  "full_sync": false,
  "skip_email": false,
  "skip_sync": false,
  "skip_report": false,
  "supplement_relations": false,
  "multi_report": false,
  "extensible_report": false,
  "ai_report": false
}
```

运行：

```bash
python3 auto_run.py --config config.json
```

### 方式2：使用命令行参数

```bash
python3 auto_run.py \
  --repo-owner matrixorigin \
  --repo-name matrixone \
  --email wupeng@matrixorigin.cn
```

### 方式3：使用环境变量

```bash
export REPO_OWNER=matrixorigin
export REPO_NAME=matrixone
export REPORT_EMAIL=wupeng@matrixorigin.cn
python3 auto_run.py
```

### 配置优先级

配置值的优先级（从高到低）：
1. 命令行参数
2. 配置文件
3. 环境变量
4. 默认值

## 命令行参数

### 基本参数

- `--config PATH`: 配置文件路径（JSON 或 YAML 格式）
- `--repo-owner OWNER`: 仓库所有者（例如：matrixorigin）
- `--repo-name NAME`: 仓库名称（例如：matrixone）
- `--email EMAIL`: 收件人邮箱

### 运行选项

- `--full-sync`: 执行全量同步（清空旧数据重新采集）
- `--skip-email`: 跳过邮件发送
- `--skip-sync`: 跳过数据同步
- `--skip-report`: 跳过报告生成
- `--supplement-relations`: 同步完成后（或与 `--skip-sync` 同时使用时）执行关联关系补全
- `--multi-report`: 额外生成多维度分析报告（客户/层级/共用 Feature/阻塞链等）
- `--extensible-report`: 额外生成可扩展分析报告（标签/模块/趋势等，配置驱动，输出 JSON/Markdown）
- `--ai-report`: 额外生成 AI 驱动分析报告（项目推进+横向关联）；**会分析 matrixone 和 matrixflow 两个仓库**，分析结果写入邮件正文，**邮件中 matrixflow 优先展示**（用户更关注 matrixflow）
- `--check-only`: 仅检查配置，不运行分析
- `--run-backup`: 流程结束后执行 **MatrixOne 数据库快照备份**（建议与同步/报告分开，作为单独定时任务；详见 [MatrixOne备份与恢复](matrixone/MatrixOne备份与恢复.md)）

## 定时事项说明（参考误删/故障恢复）

建议将以下两类事项**分开**安排，便于在误删或故障后恢复：

| 事项 | 说明 | 推荐方式 |
|------|------|----------|
| **① 同步 / 分析 / 报告 / 邮件** | 拉取 Issue、生成报告、发邮件 | 使用 `auto_run.py`（可配合 cron 或任务计划程序） |
| **② 数据库备份** | 为 MatrixOne 创建快照，便于回滚 | **单独定时**执行 `scripts/backup_matrixone.py`，或在本流程中加 `--run-backup` |

- 数据库备份**不作为**同步流程的前置/强依赖，可独立于 `auto_run.py` 定时运行（例如每日凌晨执行一次备份）。
- 备份与恢复要点见：[docs/matrixone/MatrixOne备份与恢复.md](matrixone/MatrixOne备份与恢复.md)。

## 使用示例

### 示例1：完整流程（同步 + 报告 + 邮件）

```bash
python3 auto_run.py \
  --repo-owner matrixorigin \
  --repo-name matrixone \
  --email wupeng@matrixorigin.cn
```

### 示例2：只同步数据，不生成报告和发送邮件

```bash
python3 auto_run.py \
  --repo-owner matrixorigin \
  --repo-name matrixone \
  --skip-report \
  --skip-email
```

### 示例3：只生成报告，不同步和发送邮件

```bash
python3 auto_run.py \
  --repo-owner matrixorigin \
  --repo-name matrixone \
  --skip-sync \
  --skip-email
```

### 示例4：全量同步

```bash
python3 auto_run.py \
  --repo-owner matrixorigin \
  --repo-name matrixone \
  --full-sync
```

### 示例5：同步后自动补全关联关系

```bash
python3 auto_run.py \
  --repo-owner matrixorigin \
  --repo-name matrixone \
  --supplement-relations
```

### 示例6：使用配置文件

```bash
# 创建配置文件
cat > my_config.json << EOF
{
  "repo_owner": "matrixorigin",
  "repo_name": "matrixone",
  "email": "wupeng@matrixorigin.cn"
}
EOF

# 运行
python3 auto_run.py --config my_config.json
```

### 示例7：关联补全 + 多维度报告 + 邮件（跳过同步）

```bash
python3 auto_run.py \
  --repo-owner matrixorigin \
  --repo-name matrixone \
  --skip-sync \
  --supplement-relations \
  --multi-report
```

### 示例7b：生成可扩展分析报告并发送综合分析邮件

```bash
python3 auto_run.py \
  --repo-owner matrixorigin \
  --repo-name matrixone \
  --skip-sync \
  --extensible-report \
  --email 收件人@example.com
```

使用 `--extensible-report` 且发送邮件时，**邮件正文会包含可扩展分析结果**（标签分析、模块分析、客户分析、基础统计等），而不仅是基础日报/进度报告。可先运行诊断脚本验证可扩展分析：

```bash
python3 test_analysis.py
```

或单独生成报告（不发邮件）：`python3 run_extensible_analysis.py --repo-owner matrixorigin --repo-name matrixone`

可扩展分析配置文件：`config/analysis_config.yaml`。未指定 `--email` 时使用 `config/config.py` 中的 `DEFAULT_EMAIL_TO`。

### 示例8：AI 驱动分析并发送邮件（matrixone + matrixflow，matrixflow 优先）

```bash
python3 auto_run.py \
  --repo-owner matrixorigin \
  --repo-name matrixflow \
  --skip-sync \
  --ai-report \
  --email 收件人@example.com
```

使用 `--ai-report` 时，会**同时分析** `matrixorigin/matrixflow` 和 `matrixorigin/matrixone` 两个仓库，将项目推进分析（客户健康度、关键发现、建议行动）和横向关联分析（共性需求、战略建议）写入邮件正文。**邮件中 matrixflow 排在前面**（用户更关注 matrixflow）。未指定 `--email` 时使用 `config/config.py` 中的 `DEFAULT_EMAIL_TO`。

### 示例9：仅检查配置

```bash
python3 auto_run.py --check-only
```

## 定时任务配置

**说明**：若希望定时邮件为**丰富版**（含可扩展分析、AI 分析），必须在命令中加 `--extensible-report --ai-report`，或在 `config.json` 里设置 `"extensible_report": true`, `"ai_report": true`。否则邮件仅为日报+进度简单版。

### Linux/macOS (crontab)

每天上午 9 点运行同步/报告，**单独**在每天凌晨 2 点运行数据库备份（推荐）：

```bash
# 编辑 crontab
crontab -e

# 添加以下行（根据实际路径修改）
# ① 同步/报告/邮件（每天 9 点）
# 简单版邮件（仅日报+进度）：
0 9 * * * cd /path/to/GitHub_Issue_智能管理系统 && /usr/bin/python3 auto_run.py --config config.json >> logs/auto_run.log 2>&1
# 丰富版邮件（可扩展分析 + AI 分析，推荐）：
# 0 9 * * * cd /path/to/GitHub_Issue_智能管理系统 && /usr/bin/python3 auto_run.py --repo-owner matrixorigin --repo-name matrixflow --extensible-report --ai-report --email wupeng@matrixorigin.cn >> logs/auto_run.log 2>&1

# ② 数据库备份（单独定时，每天凌晨 2 点，建议与上面分开）
0 2 * * * cd /path/to/GitHub_Issue_智能管理系统 && /usr/bin/python3 scripts/backup_matrixone.py >> logs/backup_matrixone.log 2>&1
```

若希望在一次定时中同时跑同步+备份，可在同一行加上 `--run-backup`，例如：
`0 9 * * * cd /path/to/... && python3 auto_run.py --config config.json --run-backup >> logs/auto_run.log 2>&1`

### Windows 任务计划程序

1. 打开"任务计划程序"
2. 创建基本任务
3. 设置触发器（例如：每天 9:00）
4. 设置操作：启动程序 `python3`，参数 `auto_run.py --config config.json`，起始于项目目录

### 使用 systemd (Linux)

创建服务文件 `/etc/systemd/system/github-issue-manager.service`：

```ini
[Unit]
Description=GitHub Issue Manager Auto Runner
After=network.target

[Service]
Type=oneshot
User=your_user
WorkingDirectory=/path/to/GitHub_Issue_智能管理系统
ExecStart=/usr/bin/python3 auto_run.py --config config.json
StandardOutput=append:/path/to/GitHub_Issue_智能管理系统/logs/auto_run.log
StandardError=append:/path/to/GitHub_Issue_智能管理系统/logs/auto_run.error.log
```

创建定时器文件 `/etc/systemd/system/github-issue-manager.timer`：

```ini
[Unit]
Description=Run GitHub Issue Manager Daily
Requires=github-issue-manager.service

[Timer]
OnCalendar=daily
OnCalendar=09:00
Persistent=true

[Install]
WantedBy=timers.target
```

启用定时器：

```bash
sudo systemctl enable github-issue-manager.timer
sudo systemctl start github-issue-manager.timer
```

## 环境变量配置

除了运行参数，还可以通过环境变量配置系统设置：

```bash
# GitHub Token
export GITHUB_TOKEN=your_github_token

# AI Provider (qwen, openai, claude)
export AI_PROVIDER=qwen

# 千问 API Key
export DASHSCOPE_API_KEY=your_dashscope_api_key

# 数据库配置
export DATABASE_TYPE=matrixone
export MATRIXONE_HOST=your_host
export MATRIXONE_PORT=6001
export MATRIXONE_USER=your_user
export MATRIXONE_PASSWORD=your_password
export MATRIXONE_DATABASE=github_issues

# 邮件配置
export SMTP_HOST=smtp.qq.com
export SMTP_PORT=587
export SMTP_USER=your_email@qq.com
export SMTP_PASSWORD=your_smtp_password
export EMAIL_FROM=your_email@qq.com
export EMAIL_FROM_NAME=GitHub Issue Manager
```

## 故障排查

### 1. 配置检查失败

运行 `--check-only` 检查配置：

```bash
python3 auto_run.py --check-only
```

### 2. 同步失败

- 检查 GitHub Token 是否正确
- 检查网络连接
- 检查仓库名称是否正确

### 3. 邮件发送失败

- 检查 SMTP 配置是否正确
- 检查邮箱授权码（不是登录密码）
- 检查防火墙是否阻止 SMTP 端口

### 4. 报告生成失败

- 检查数据库连接
- 检查是否有数据（需要先同步）
- **多维度报告失败**：若出现 SQL 子查询相关错误，已通过「先查 latest_time 再传参」修复；多维度报告失败不会阻断邮件发送，基础报告仍会发出

## 注意事项

1. **全量同步**：`--full-sync` 会清空所有旧数据，请谨慎使用
2. **API 限制**：GitHub API 有速率限制，大量 Issue 可能需要较长时间
3. **邮件配置**：某些邮箱服务商需要开启 SMTP 服务并获取授权码
4. **数据库连接**：确保数据库服务正常运行且网络可达
