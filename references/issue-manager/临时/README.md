# 临时文件目录

本目录用于存放调试、测试过程中产生的临时文件，以及一次性的辅助脚本/报告。

## 用途

- **调试报告**：连接测试、配置检查、分析测试等生成的报告
- **临时脚本**：一次性使用的辅助脚本（如 `sync_missing_issues.py`、`运行并输出.sh`）
- **过程文档**：开发调试过程中产生的说明、对比分析等

## 使用规则

- 今后新生成的临时文件统一放在此目录
- 已运行且不再需要的脚本可在此目录删除，避免根目录杂乱
- 根目录保持为项目核心代码和正式文档

## 当前归档内容

- 各类测试/诊断报告（`*报告.md`、`*说明.md`）
- 一次性辅助脚本：`sync_missing_issues.py`、`运行并输出.sh`
- **出图测试**：`report_to_image.py` — 把分析结果 HTML 转成固定宽长图，先在此改着测，再考虑并入主项目；出图目录 `output_images/`
