# GitHub数据拉取测试说明

## 📋 测试脚本说明

我为您创建了两个测试脚本，用于测试从GitHub拉取Issues和关联关系：

### 1. `test_github_simple.py` - 简单快速测试（推荐）

**功能：**
- 快速测试获取Issues列表
- 测试获取Issue评论
- 测试提取Issue之间的关联关系

**使用方法：**

**方式1：修改脚本中的仓库信息**
```python
# 在 test_github_simple.py 中找到这两行，修改为您的仓库：
repo_owner = "wupeng"  # 修改为您的GitHub用户名
repo_name = "test-repo"  # 修改为您的仓库名
```

然后运行：
```bash
cd "/Users/wupeng/Desktop/GitHub_Issue_智能管理系统"
python3 test_github_simple.py
```

**方式2：通过命令行参数指定**
```bash
python3 test_github_simple.py <GitHub用户名> <仓库名>
```

示例：
```bash
python3 test_github_simple.py wupeng my-repo
```

### 2. `test_github_fetch.py` - 完整详细测试

**功能：**
- 完整的测试流程
- 详细的输出信息
- 保存测试结果到JSON文件

**使用方法：**
```bash
python3 test_github_fetch.py
```

运行后会交互式询问：
- 仓库所有者（GitHub用户名）
- 仓库名称
- 要测试的Issues数量

## 🎯 测试内容

两个脚本都会测试以下功能：

1. **获取Issues列表**
   - 从GitHub API获取指定仓库的所有Issues
   - 显示Issue的基本信息（编号、标题、状态、创建时间等）

2. **获取Issue评论**
   - 获取指定Issue的所有评论
   - 显示评论作者、时间和内容

3. **提取关联关系**
   - 从Issue正文和评论中提取对其他Issue的引用
   - 识别关系类型：mention（提及）、fixes（修复）、duplicate（重复）、related（相关）、blocks（阻塞）、depends_on（依赖）等
   - 显示关联的上下文信息

## ⚙️ 配置要求

确保 `config/config.py` 中已正确配置GitHub Token：

```python
GITHUB_TOKEN = "ghp_xxxxxxxxxxxxxxxxxxxx"  # 您的GitHub Token
```

## 🔍 关联关系识别规则

脚本会自动识别以下类型的关联关系：

- **mention（提及）**: 在正文或评论中提到了其他Issue（如 `#123`）
- **fixes（修复）**: 包含 "fixes"、"fix"、"解决"、"修复" 等关键词
- **duplicate（重复）**: 包含 "duplicate"、"重复"、"相同" 等关键词
- **related（相关）**: 包含 "related"、"相关"、"关联" 等关键词
- **blocks（阻塞）**: 包含 "blocks"、"阻塞"、"阻止" 等关键词
- **depends_on（依赖）**: 包含 "depends on"、"依赖"、"需要" 等关键词

## ⚠️ 注意事项

1. **API限流**: GitHub API有速率限制，如果请求过多可能需要等待
2. **Token权限**: 确保您的GitHub Token有足够的权限访问目标仓库
3. **网络连接**: 需要能够访问GitHub API（api.github.com）
4. **仓库权限**: 只能访问您有权限访问的仓库

## 🐛 常见问题

### 1. 提示"配置验证失败"
- 检查 `config/config.py` 中的 `GITHUB_TOKEN` 是否已设置
- Token格式应为：`ghp_xxxxxxxxxxxxxxxxxxxx`

### 2. 提示"404 Not Found"
- 检查仓库所有者用户名和仓库名是否正确
- 确认您有权限访问该仓库

### 3. 提示"API限流"
- 等待一段时间后重试
- 脚本会自动处理限流，等待后重试

### 4. 没有找到关联关系
- 这是正常的，如果Issue中没有引用其他Issue（如 `#123`），就不会有关联关系
- 可以在Issue正文或评论中添加对其他Issue的引用来测试
